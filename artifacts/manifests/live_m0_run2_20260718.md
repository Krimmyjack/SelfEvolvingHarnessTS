# Live M0 Run 2: valid negative at the controller boundary

Run directory: `runs/minipipe/live-scientific-20260718-05`. Raw prompts,
responses, private feedback, and caches remain ignored. This archive contains no
secret material.

## Result

- The real GPT-5.5/Chronos run completed two cycles without transport failure.
- 127 of 135 Agent responses were valid envelopes; all seven slow responses
  were valid envelopes.
- The no-op case rate fell from Run 1's 12/36 to 1/36.
- Level observables routed five of six damaged level-shift targets to
  `SKILL_LIBRARY_GAP`.
- Three of seven slow outputs could be constructed as `EditManifest` objects,
  up from 0/2 in Run 1.
- No manifest passed controller preflight, no paired replay ran, and H0 remained
  active in both cycles. Cycle 1 therefore reproduced cycle 0 from cache and
  added no scientific information.

The preregistered primary and stretch outcomes both failed. This is a valid
negative result: it proves that Run 1's repairs worked while identifying the
next boundary precisely.

## Causal ledger

The dominant failure is a contract gap. The declared slow schema accepted
arbitrary PATCH objects, free-form behavior predictions, and arbitrary ADD
payloads while `EditManifest` and the controller enforced closed forms. Four
PATCH proposals died at construction and three ADD proposals died at the
closed behavior-predicate check.

The model has a separate protocol ledger: eight fast responses violated the
outer-envelope contract, and the three ADD proposals reused the identity of an
existing bootstrap skill instead of defining a learned capability. These do
not erase the schema/controller mismatch; they require bounded protocol retry
and explicit inventory/entry contracts.

## Launch gate

No full Run 3 is admissible until all of the following pass:

1. schema-valid manifests imply success at the controller's static shape layer;
2. public-safe correctable errors receive at most one retry;
3. the probe InstrumentPatch is calibrated and assigned a new instrument epoch;
4. a public-only reference level edit passes offline apply/compile/retrieval;
5. a bounded live wind tunnel demonstrates the chain
   `retrieved -> supplied -> selected -> G>0 -> risk stable`.

The reference edit and wind tunnel are calibration assets. Their success must
not be counted as autonomous Harness growth.
